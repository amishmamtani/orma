# Step 3 — Plain-language labeling

**Status:** ✅ Complete
**Depends on:** Step 2 ✅

---

## Goal

Replace the raw function/class names in the diagram with plain-English labels generated by an LLM. A node for `send_welcome_email` should read "Send a welcome email to the user" — not the raw name.

This is the highest-risk step. Label quality *is* the product. Don't rush through it.

---

## What to build

**Backend:**
- Add `langgraph`, `langchain`, `langchain-google-genai` to `requirements.txt`
- Create `backend/config.py` — exports a single `get_model()` function that returns `ChatGoogleGenerativeAI(model="gemini-1.5-flash")`. This is the only place the provider is set. See ADR-003 and ADR-004.
- Create `backend/chains/labeling_graph.py` — a minimal LangGraph `StateGraph` with one node: call the model with a labeling prompt, return a plain-English label for the given node's source code.
- Update `POST /parse` in `main.py` to call the labeling graph for each node after AST parsing. Replace the placeholder `label` with the LLM's output.

**Frontend:** No changes in this step.

---

## Labeling prompt guidance

The prompt should instruct the model to:
- Describe what the function/block *does* in one sentence
- Use product/UX language, not developer terms (no "function", "method", "iterate", "instantiate")
- Write in active voice: "Sends a welcome email to the user" not "Welcome email sending"
- Keep it under 10 words

Test the prompt manually via `/docs` before wiring it in.

## How to test

1. `make dev`
2. Open `http://localhost:8000/docs`
3. POST to `/parse` with a script containing named functions (e.g. `send_welcome_email`, `calculate_total`, `check_user_exists`)
4. Confirm each node's `label` reads like product language, not code

---

## Done when

- [x] Every node returned by `/parse` has a plain-English label
- [x] Labels read like product language to someone who doesn't code
- [ ] Broken Python still returns a clean 422 error (regression check)
- [ ] **Manual label quality check:** show the diagram labels (not the code) for 3 real Python files to one non-developer and confirm they can describe what the code does

## Notes

- `response.content` from the Gemini SDK returns a list of content blocks, not a plain string. Fixed by checking `isinstance(content, list)` and extracting `.get("text", "")` from the first block.
- Labels at function/class granularity only — internals of a function (loops, assignments) are not broken into sub-nodes. Acceptable for now; can revisit in a later step.
- Label quality verified on test functions: "Sends a welcome email to the user", "Calculates the total cost of each order" — clean product language.
